Two separate things to address — the secret update is easy, the App Store build takes a few steps.

Updating the Supabase URL secret
You can't overwrite a secret from the terminal — Replit secrets are managed through the UI only. Here's how:

Click the 🔒 Secrets tab in the left sidebar
Find EXPO_PUBLIC_SUPABASE_URL
Click on it → edit the value → Save
That's it. No terminal command needed (and no command exists to do it — the terminal can't read or write your secrets).

TestFlight + App Store build
This requires EAS Build (Expo's cloud build service). The full flow is:

One-time setup (on your Mac or in a terminal with your Apple credentials)
npm install -g eas-cli
eas login                          # your Expo account
eas build:configure                # adds eas.json
eas credentials                    # links your Apple Developer Team + certificates

Build for TestFlight
eas build --platform ios --profile preview

EAS builds the native binary in the cloud (no Xcode needed locally) and gives you an .ipa file.

Submit to TestFlight
eas submit --platform ios --latest

Uploads to App Store Connect. Then in App Store Connect, add your testers to the closed beta group.

For the Watch app
The Watch target has to be added in Xcode before running eas build. This means:

expo prebuild → generates the ios/ folder
Open ios/Judith.xcworkspace in Xcode
Add a WatchKit App target
Then run eas build

The Judith app lives inside a subfolder of the main project. On your Mac, wherever you have this project saved, you need to go into artifacts/judith:

cd ~/path-to-your-project/artifacts/judith
eas build --platform ios --profile preview

To find it quickly, run this to locate app.json:

find ~ -name "app.json" -path "*/judith/*" 2>/dev/null

That'll print the exact full path. Then just cd to that folder and your EAS commands will work.
