# WATCH_APP.md — Judith for Apple Watch (watchOS)

A companion watchOS app so users glance, get reminded, and **mark a bill paid from the wrist** without opening the phone. Judith is a **tracker** — "Mark paid" records payment, never moves money. Built in the prototype as a viewable concept: **Settings → "Apple Watch app"** (source: `source/japp-watch.jsx` + watch CSS in `source/Judith - Full Flow.html`, search `.wf-bezel` / `.wos-`).

## Build with
- **SwiftUI + WidgetKit (watchOS)** as a companion target in the same Expo/RN project (RN doesn't run on watchOS — the watch app is native Swift, sharing data with the phone app via an **App Group** + the same Supabase session/token, or a phone↔watch sync via WatchConnectivity).
- Data source: the user's `bill_cycles` (next due, totals, status). Mark-paid on the watch writes back through the shared store / WatchConnectivity → Supabase.

## Design tokens (carry from the phone app)
- Pure black background `#000` (OLED — saves battery, matches watch convention), surfaces `#181b22 / #1f232c`, accent mint `oklch(0.78 0.15 168)` → use the precomputed RN/Swift hex.
- Numbers in a tabular/mono face; urgency colors overdue/urgent/near/ok identical to phone.
- Big tap targets (watch HIG: full-width rounded buttons), Digital-Crown scrolling for lists.
- Screen proportions ≈ Series 45mm (396×484pt). The prototype frames are 198×240 (½-scale) with a 42px-radius screen.

## Screens (6 — each is a reference frame in the prototype)

### 1. Watch face complication (Modular / Infograph)
Judith as the **large complication**: "NEXT DUE · {n}D", provider + amount (amount in `--near`), and a bottom row "{count} due · {total}". Tapping opens the app. Also offer small corner complications: due-count ring, month-total.

### 2. Reminder (long-look notification)
Fires `leadDays` before due (default 3d, 9:00 AM). Header: Judith avatar + "Judith" + "now". Title "{provider} due in {n} days", amount big in `--near`. **Inline actions: "Mark paid" (primary, mint) + "Remind tomorrow."** Persona-flavored title copy allowed (match the phone reminders spec).

### 3. App — Up next (list)
Header "Up next" + month total. Scrollable rows (Digital Crown): urgency dot + provider + "in {n}d" / "{n}d late" + amount. Overdue rows use `--overdue`. Tap a row → detail.

### 4. Bill detail
Category cap + urgency dot, provider, big amount, "Due in {n} days" (or "{n}d ago" if overdue). **Actions: "Mark paid" (primary) + "Snooze 1 day."**

### 5. Marked-paid confirmation
Big mint check, "Marked paid", "{provider} · {amount}", and a **streak nudge** "🔥 {streak}-mo streak" (ties to the retention/track-record system). Brief, then auto-returns to the list.

### 6. Smart Stack widget
A surfacing widget when a bill is near: "JUDITH" + provider + amount + "due in {n} days · tap to pay". Uses WidgetKit relevance so it rises in the Smart Stack as the due date approaches.

## Behavior notes
- **Mark paid** on the watch = same `markPaid` semantics as the phone (records `paidVia` if chosen on phone; watch defaults to "Marked paid"), advances to the next reminder, updates streak/fees-avoided.
- **Snooze** = +1 day on the reminder, same as phone.
- **Complication timeline:** refresh on data change + a daily reload; show the single most-urgent unpaid bill.
- **Voice:** on Series with a speaker, Judith's reminder can be spoken (English, chosen ElevenLabs voice) — optional, post-MVP.
- **Offline:** the watch shows last-synced data and queues a mark-paid to sync when reconnected.

## Acceptance

- ☐ Reminder long-look with working Mark paid / Remind tomorrow
- ☐ Up-next list scrolls with the Crown; overdue rows red; tap → detail
- ☐ Detail Mark paid / Snooze work and sync to phone/Supabase
